## 3. Roadmap


### Phase 1 — Close the core & go live (0–3 mo)

- Deploy ROOT Assistant + booking/OTP automation and wire a WhatsApp Business (BSP) provider — directly attacks the #1 pain (slow / after-hours replies).
- Activate live messaging outbox: booking confirmation, pre-arrival directions, payment request/reminder templates + triggers.
- Configurable cancellation policy (free 4 days / peak 2 days) + refund workflow (request, approve, partial, status).
- Guest fields: address, vehicle number, preferences/tags; activate ID upload + ID-verified/checked flags + retention.
- Complaints module (category, priority, status, assignment, resolution, CSAT follow-up).
- Online payment collection (UPI/Razorpay link) + pending-payments dashboard card.
- Data migration tool (paper/WhatsApp → guests/bookings); add missing channels (Instagram, FB, travel agent, walk-in, word-of-mouth); activate iCal + email forwarder.

### Phase 2 — Complete the PMS & team (3–7 mo)

- Multi-tenancy + real auth and role-based access (reception sees bookings, not finance) — the gating epic.
- Staff directory, shift scheduling, attendance; housekeeping assignment + checklist + accountability.
- Maintenance module + asset register + preventive maintenance; Inventory + low-stock alerts/reorder.
- Vendor & procurement module; Transport trips/driver records in OTA (dispatch stays in ROOT CabAgent).
- Group & long-stay bookings; amenities/facilities model; reviews tracker + ROOT review-request automation.
- Audit log + guest-consent capture; plain-language ROOT Console over analytics/finance.

### Phase 3 — Regional community network ('Rapido for homestays') (7–12 mo)

- Overflow referral marketplace: recipient property, conversion, revenue, expected favour, credit balance, referral analytics.
- Shared live availability + searchable property directory (pets, wheelchair, big groups, parking, airport pickup).
- Community scam network (export + verified, moderated sharing) and evidence-backed bad-guest alerts with review/appeal safeguards.
- No-show reliability score → shared repeat-offender flag; shared trusted vendors/drivers/guides; emergency assistance.
- Offline tolerance for patchy connectivity; multi-location rollout.
